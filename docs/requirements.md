# VPS IP 封锁原因投票统计网站 — 需求文档

## 一、项目目标

通过投票收集 VPS 用户的 IP 封锁数据，按厂商、ASN、用途、协议、配置等维度统计封锁概率，生成旭日图可视化和 AI 分析报告。

## 二、数据模型

每条投票记录代表**一组相同配置的 VPS 机器**：

| 字段 | 必填 | 说明 |
|------|------|------|
| is_blocked | 是 | 是否被封（被封 / 未被封） |
| org | 是 | 厂商名称，如 Vultr |
| asn | 是 | AS 号码，如 AS20473（同一厂商可能有多个 ASN） |
| usage | 是 | 用途（代理 / 网站） |
| proxy_protocol | 用途=代理时 | 代理协议 |
| key_config | 用途=代理时 | 关键配置特征（单选） |
| count | 是 | 相同配置的机器数量（默认1） |

### 数据层级结构

投票数据按以下层级组织（也是旭日图的层级）：

```
是否被封 → 厂商(Org) → ASN → 用途 → 协议 → 关键配置
```

- 用途=网站时，协议和配置层不适用，填 `-`。

> ASN 本身包含地区归属信息，无需单独设置地区字段。

### 厂商/ASN 获取方式

用户填写厂商和 ASN 时，提供两种方式（二选一）：

1. **输入 IP 自动获取（推荐）** — 后端调用 IP 查询服务获取厂商和 ASN，用户确认后继续。**IP 不存库**，仅用于查询。
2. **从预设列表选择** — 先选厂商，再选该厂商下的 ASN。也支持自定义输入（走动态选项机制）。

## 三、预设选项

### 第1层：是否被封

- 被封 / 未被封

### 第2层：厂商（Org）

预设常见 VPS 厂商：

- Vultr、Hostinger、OVH、DigitalOcean、Linode (Akamai)
- AWS、Azure、GCP
- 阿里云、腾讯云
- 搬瓦工 (IT7)、RackNerd (HostPapa)、DMIT、CloudCone

> 支持自定义输入，走动态选项机制。

### 第3层：ASN（厂商下属）

用户选择厂商后，选择对应的 ASN。预设厂商与 ASN 的映射关系：

| 厂商 | ASN |
|------|-----|
| Vultr | AS20473 |
| Hostinger | AS47583 |
| OVH | AS16276 |
| DigitalOcean | AS14061 |
| Linode (Akamai) | AS63949 |
| AWS | AS14618 |
| Azure | AS8075 |
| GCP | AS15169 |
| 阿里云 | AS45102 |
| 腾讯云 | AS132203 |
| 搬瓦工 (IT7) | AS25820 |
| RackNerd (HostPapa) | AS36352 |
| DMIT | AS54574 |
| CloudCone | AS54643 |

> 同一厂商可能有多个 ASN，支持自定义输入，走动态选项机制。

### 第4层：用途

- 代理 / 网站

### 第5层：代理协议（用途=代理时）

| 分类 | 协议 |
|------|------|
| Shadowsocks 系列 | Shadowsocks、ShadowsocksR、Shadowsocks-2022 |
| VMess 系列 | VMess+TCP、VMess+WS+TLS、VMess+gRPC+TLS、VMess+HTTP/2 |
| VLESS 系列 | VLESS+TCP+TLS、VLESS+WS+TLS、VLESS+gRPC+TLS、VLESS+XTLS-Vision、VLESS+Reality+Vision、VLESS+Reality+gRPC |
| Trojan 系列 | Trojan+TLS、Trojan+WS+TLS、Trojan+gRPC+TLS、Trojan+Reality |
| QUIC/UDP 协议 | Hysteria、Hysteria2、TUIC v5、Juicity |
| VPN 协议 | WireGuard、AmneziaWG、OpenVPN、IPSec/IKEv2 |
| 伪装/混淆类 | SS+Shadow-TLS v3、NaiveProxy、Brook、Snell |
| 其他 | SSH 隧道、SoftEther VPN、Mieru |

> 支持自定义输入，走动态选项机制。

### 第6层：关键配置（协议的下一层，旭日图最外层）

用户选择一项最关键的配置特征：

| 分类 | 选项 |
|------|------|
| TLS 伪装 | TLS 指纹伪装为 Chrome、TLS 指纹伪装为 Firefox |
| CDN 中转 | 使用 Cloudflare CDN 中转、使用其他 CDN 中转 |
| 端口策略 | 使用 443 端口、使用非常规端口、端口跳跃（Port Hopping） |
| 连接方式 | 直连无伪装、开启 Mux 多路复用 |
| 使用人数 | 独享（仅自己）、少量共享（2-5人）、多人共享（6-20人）、机场节点（>20人） |

> 支持自定义输入，走动态选项机制。

## 四、动态选项机制

处理用户自定义输入的新选项，防止重复并控制质量。

### 核心规则

1. 用户在任意层级可输入自定义选项
2. 系统对自定义输入进行**智能去重**（识别多语言、缩写、俗称、大小写变体等等价表述）
3. 若与已有选项等价 → 归入已有选项
4. 若为全新选项 → 创建新选项，记录提交计数
5. 累积 ≥3 个不同用户提交同一新选项 → 升级为正式选项，出现在前端选项列表中
6. 未达标选项不显示在选项列表中，也不参与统计和可视化

### 等价判断示例

- 多语言表述：日本 = JP = Japan
- 缩写/简称：搬瓦工 = BWG = BandwagonHost
- 俗称/别名：DMIT = 大妈IT
- 大小写变体：vultr = Vultr = VULTR

### 选项列表更新

- 前端选项列表动态加载，包含：预设选项 + 已升级的自定义选项
- 新升级的选项自动出现在后续用户的投票表单中，无需手动更新

### 投票可见性规则

- 全部选择预设选项或已升级选项的投票 → 立即参与统计
- 含自定义输入的投票 → 等待系统处理后决定：
  - 匹配到已有选项 → 参与统计
  - 创建了未达标的新选项 → 暂不参与统计
- 当新选项升级时，之前引用该选项的投票也一并加入统计

## 五、可视化与输出

### 5.1 旭日图

- 从中心向外逐层展开：是否被封 → 厂商 → ASN → 用途 → 协议 → 配置
- 支持点击钻取下一层
- 每个扇区显示：**数量**、**占父级比**（同层分布）、**占总比**（全局概率）
- 页面显示"数据更新于 X 分钟前"，让用户知道数据时效

### 5.2 Markdown 扁平表导出

导出一张完整的扁平表格，每行是一条从根到叶的完整路径：

```markdown
总投票数: 1000 台

| 是否被封 | 厂商 | ASN | 用途 | 协议 | 关键配置 | 数量 | 占总比 |
|----------|------|-----|------|------|----------|------|--------|
| 被封 | 搬瓦工 | AS25820 | 代理 | Shadowsocks | 直连无伪装 | 20 | 2.0% |
| 被封 | 搬瓦工 | AS25820 | 代理 | VMess+WS+TLS | CF CDN中转 | 25 | 2.5% |
| 未被封 | Vultr | AS20473 | 网站 | - | - | 30 | 3.0% |
| 未被封 | Vultr | AS20473 | 代理 | VLESS+Reality | TLS伪装Chrome | 18 | 1.8% |
```

**排列规则：** 按层级从左到右排序，同层内按数量降序。

**比例说明：**
- **占总比** = 当前数量 / 总投票数（看全局概率）

### 5.3 AI 分析报告

将导出表格输入 AI，自动生成分析内容：

- 封锁率最高的组合路径
- 同配置被封/未被封对比
- 高风险因素识别
- 建议和结论

**触发方式：** 用户在报告页面手动点击"生成报告"按钮（不自动生成）。

**过期提示：** 生成后记录时间戳和当时总投票数，页面展示"上次生成：2小时前 | 此后新增 42 条投票"，帮助用户判断是否需要重新生成。

## 六、投票流程

```
1. 选择"被封"或"未被封"
2. 提供厂商信息：输入 IP 自动获取 或 从列表选择厂商
3. 选择该厂商下的 ASN
4. 选择用途：
   - 代理 → 选协议 → 选关键配置
   - 网站 → 结束
5. 输入相同配置机器数量（默认1）
6. 提交（可继续添加下一组不同配置的机器）
```

## 七、防刷票规则

- 同一设备限制投票频率（基于浏览器指纹）
- 单 IP 每小时投票次数上限
- 不设账号体系，降低用户参与门槛，用技术手段替代
