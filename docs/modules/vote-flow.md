# 投票流程模块

> 投票提交的完整流程和 resolved 状态管理。涉及文件：`src/app/api/vote/route.ts`、`src/lib/validators.ts`、`src/lib/vote-service.ts`（`tryResolveVote` 等业务逻辑）

## 一、投票提交流程

```
1. 选择"被封"或"未被封"
2. 提供厂商信息：输入 IP 自动获取 或 从列表选择
3. 选择该厂商下的 ASN
4. 选择用途：
   - 代理 → 选协议 → 选关键配置
   - 网站 → 跳过协议和配置
5. 输入相同配置机器数量（默认1）
6. 提交（可继续添加下一组不同配置的机器）
```

## 二、后端处理流程

1. **校验** — Zod 验证请求数据，确保 `usage=website` 时 `protocol` 和 `keyConfig` 为 `null`（必须显式传 `null`，不能省略）
2. **速率限制检查** — 检查 IP + fingerprint 维度和 IP 维度的限流（见 rate-limit.md）
3. **指纹校验** — 验证 fingerprint 格式和防伪规则（见 rate-limit.md）
4. **入库** — 创建 Vote 记录
5. **字段分类** — 逐字段判断是否为已知选项（`isPreset=true` 或 `promoted=true`），查询需包含 `parentKey` 精确匹配
6. **确定 resolved 状态：**
   - 全部已知 → `resolved=true`
   - 含自定义字段 → `resolved=false`，为每个自定义字段入队 AI 匹配任务（见 dynamic-options.md）
7. **返回** — 立即返回成功，不等待 AI 匹配完成

## 三、resolved 状态管理

`resolved` 控制投票是否参与统计。所有聚合查询只统计 `resolved=true` 的投票。

### tryResolveVote 逻辑

通用的 resolve 检查函数，逐字段检查对应的 DynamicOption 是否为预设或已升级：
- 所有字段都是预设/已升级 → 设为 `resolved=true`
- 任一字段引用了未升级选项 → 保持 `resolved=false`

### 调用时机

1. **AI 队列处理完一条 Vote 的自定义字段后** — 更新字段值为标准名称，然后检查 resolve
2. **某 DynamicOption 升级后** — 查找所有引用该选项且 `resolved=false` 的投票，逐条检查 resolve

### 多字段自定义场景

一条 Vote 同时自定义了 org 和 protocol：两个字段分别在各自的队列中处理。只有当两个字段对应的选项都升级后，该投票才会 resolve。

## 四、IP 查询服务

- 用户输入 IP → 后端调用 ip-api.com 获取 org/ASN → 返回给前端填入表单
- **IP 不存库**，仅用于查询
- 查询结果 Redis 缓存，key: `ip:{ip_address}`，TTL = 1 小时
- 主服务 ip-api.com（45次/分钟），备用 ipinfo.io（50k次/月）
- 返回的原始值（如 `Vultr Holdings LLC`）不做名称映射，直接填入输入框，提交时走标准 AI 归类流程

## 五、前端交互规范

### 表单模式

采用**分步向导（Wizard）模式**，每步一个选择，逐步推进：

1. 步骤 1：选择是否被封（两个按钮）
2. 步骤 2：提供厂商（Tab 切换：IP 查询 / 列表选择）
3. 步骤 3：选择 ASN（依赖步骤 2 的 org 值动态加载）
4. 步骤 4：选择用途（两个按钮）
5. 步骤 5（仅代理）：选择协议（下拉列表 + 自定义输入框）
6. 步骤 6（仅代理）：选择关键配置（下拉列表 + 自定义输入框）
7. 步骤 7：输入机器数量（数字输入框，默认 1）
8. 确认提交

### 验证时机

- **即时验证**：每步完成后立即校验当前步骤的字段
- 未通过验证不允许进入下一步
- 提交前不再二次校验（各步已校验过）

### IP 查询交互

- 用户输入 IP 后点击"查询"按钮
- 显示加载状态（spinner）
- 查询成功：展示返回的 org/ASN/country/city 信息，用户点击"确认使用"后填入表单
- 查询失败：显示错误提示，提供"手动选择"链接切换到列表选择模式

### 自定义输入交互

- 下拉列表末尾显示"其他（自定义输入）"选项
- 选中后展开文本输入框
- 提交时提示用户：自定义选项需等待系统审核后才能参与统计

### 错误处理 UI

| 错误码 | 前端行为 |
|--------|----------|
| 400（参数校验失败） | 表单内联显示具体字段错误信息 |
| 429（频率超限） | Toast 提示"操作过于频繁，请 {retryAfter} 秒后重试"，禁用提交按钮倒计时 |
| 502/504（服务不可用） | Toast 提示"服务暂时不可用，请稍后再试"，显示重试按钮 |
| 网络错误 | Toast 提示"网络连接失败，请检查网络后重试" |

### "继续添加"交互

- 提交成功后显示结果（resolved 状态）和两个按钮："继续添加" / "查看统计"
- "继续添加"重置表单到步骤 1，但保留上次选择的厂商和 ASN（常见场景：同厂商不同配置）
- 用户可手动修改保留的字段
